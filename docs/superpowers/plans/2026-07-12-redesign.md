# AI Playbook Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the redesign specified in `docs/superpowers/specs/2026-07-12-redesign-design.md`: new token system (white/ink/salmon/star-blue, Hanken Grotesk), rail-focus-tray boards, gate-bar-only primary actions, inline chat, checklist generating screens, editorial Review, poster Big Move, delete-undo, mobile.

**Architecture:** The app styles itself through semantic CSS classes in `src/index.css` (3,134 lines) whose colors come from a Tailwind 4 `@theme` block, mirrored by a JS object `C` in `src/config/constants.js` (used for SVG icon props). Task 1 redefines the values behind the EXISTING variable names, which reskins every screen at once with near-zero structural risk; later tasks restructure views one at a time. The reference for every visual decision is `Mock Ups/ai-playbook-miro-restructured-mockup.html` (open it in a browser; it contains the canonical CSS values).

**Tech Stack:** React 19, Vite 7, Tailwind CSS 4 (`@theme` in `src/index.css`, no config file), lucide-react icons, docx + file-saver for Word export, `window.print()` for PDF. No test runner exists; verification = `npm run build` + `npm run lint` + browser checks against the mockup.

## Global Constraints

- Branch `feat/redesign-2026-07`; tag `pre-redesign` on master first. Never push to master; PR at the end.
- `npm run build` must pass before EVERY commit (CLAUDE.md rule).
- Phase names are locked: "AI Use Cases", "Change Strategy". Board items are "categories"/"ideas" and "rules"/"actions". Never "Primitives", never "territories" in UI text.
- All user-facing copy is unchanged unless a task explicitly says otherwise.
- No green anywhere, including toasts. Red only via `--color-danger` for destructive/error.
- Colors exist in TWO places that must stay in sync: `@theme` in `src/index.css` AND the `C` object in `src/config/constants.js`.
- `.gate-bar` must remain a DIRECT child of the scroll container `.canvas-rules` (sticky pitfall; see comment in PrimitivesView.jsx line ~98). Never apply `transition: all` to `.canvas-rules`.
- Animations 0.3-0.5s, transform/opacity only. `prefers-reduced-motion` respected for new animations.
- max_tokens, prompts, api/ files: untouchable in this plan.
- Commits end with: `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>` (CLAUDE.md still says "Opus 4.6"; Task 14 updates that stale line).
- Verification pattern for every task, referred to below as **BUILD+LOOK**: run `npm run build` (expect exit 0, "built in Ns"), `npm run lint` (expect exit 0), then `npm run dev` and check the listed screens at `http://localhost:5173` against the matching mockup section.       </div>

            {g.items.map((it) => (
              <div key={it.id} className="tray-item">
                <span className="tray-item-star">★</span>
                <span>{it.text}<small>{it.sourceLabel}</small></span>
              </div>
            ))}
